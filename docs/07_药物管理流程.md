# Workflow — 药物管理流程

> 版本: v1.0  |  日期: 2025-05-29

---

## 1. 概述

药物管理是药房机械臂的业务核心，覆盖从入库→存储→分拣→出库→补仓的全生命周期。系统自动维护库存数据库，在药物低于安全阈值时触发告警。

## 3. 分步骤详解

### 3.1 药物入库

```
Step 1: 扫码
  → 管理员用条码扫描器扫描药物包装条码
  → pyzbar 解码获取药品编号

Step 2: 信息查询
  → 药品编号查询药物知识库
  → 获取药品名称、类别、用法等元数据

Step 3: 入库登记
  → 录入库存表: drug_id, name, quantity, shelf_location, batch_no,
                expiry_date, timestamp
  → 如药品已存在，则累加数量

Step 4: 货架分配
  → 基于药品类别自动分配货架区域
  → A区: 抗生素类  B区: 解热镇痛类  C区: 心血管类 ...
  → 更新 shelf_location
```

### 3.2 药物分拣

```
Step 1: 接收分拣指令
  → 来源: AI Agent 药物推荐结果
  → 数据: {drug_id, drug_name, dosage, quantity}

Step 2: 位置检索
  → drug_id → 查询库存数据库 → shelf_location
  → 生成目标位姿 (x, y, z, pitch)

Step 3: 视觉核验
  → 摄像头拍摄目标货架
  → YOLOv8 定位药盒
  → PaddleOCR 读取药名验证
  → pyzbar 扫描条码二次核验
  → 核验通过 → 计算3D位姿
  → 核验失败 → 告警，等待人工介入

Step 4: 抓取执行
  → 3D位姿 → ROS MoveIt 逆解
  → 六轴轨迹生成
  → STM32 舵机驱动
  → 夹爪抓取

Step 5: 放置
  → 搬运到出药区
  → 机械臂归位
```

### 3.3 库存更新

```sql
-- 分拣完成后自动更新库存
UPDATE drug_inventory 
SET 
  stock_quantity = stock_quantity - {sorted_quantity},
  last_updated = NOW()
WHERE drug_id = '{drug_id}';

-- 记录分拣日志
INSERT INTO sorting_log 
  (drug_id, drug_name, quantity, operator, timestamp, status)
VALUES 
  ('{drug_id}', '{drug_name}', {quantity}, 'AI-Agent', NOW(), 'completed');
```

### 3.4 补仓告警

```python
def check_stock_alert():
    """分拣完成后触发库存检查"""
    low_stock = db.query("""
        SELECT drug_id, name, stock_quantity, alert_threshold
        FROM drug_inventory
        WHERE stock_quantity <= alert_threshold
    """)
    
    for item in low_stock:
        alert_message = (
            f"⚠️ 药物库存不足\n"
            f"药品: {item.name}\n"
            f"当前库存: {item.stock_quantity}\n"
            f"安全阈值: {item.alert_threshold}\n"
            f"建议立即补货"
        )
        # 推送告警到 QQ / 短信 / 管理后台
        send_alert(alert_message)
```

## 4. 库存数据库设计

### 表: drug_inventory

| 字段 | 类型 | 说明 |
|------|------|------|
| drug_id | VARCHAR(32) PK | 药物唯一编号 |
| name | VARCHAR(128) | 药品名称 |
| generic_name | VARCHAR(128) | 通用名 |
| category | VARCHAR(64) | 药品类别 |
| stock_quantity | INT | 当前库存数量 |
| alert_threshold | INT | 补仓告警阈值 |
| shelf_location | VARCHAR(16) | 货架位置 (如 A-03-12) |
| batch_no | VARCHAR(64) | 批号 |
| expiry_date | DATE | 有效期 |
| unit | VARCHAR(16) | 单位 (盒/瓶/片) |
| created_at | DATETIME | 入库时间 |
| last_updated | DATETIME | 最后更新时间 |

### 表: sorting_log

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK AUTO | 日志ID |
| drug_id | VARCHAR(32) FK | 药物编号 |
| drug_name | VARCHAR(128) | 药品名称 |
| quantity | INT | 分拣数量 |
| operator | VARCHAR(64) | 操作者 (AI-Agent / 管理员) |
| status | VARCHAR(16) | completed / failed / cancelled |
| error_msg | TEXT | 失败原因 |
| timestamp | DATETIME | 操作时间 |

## 6. 异常处理

| 异常场景 | 处理策略 |
|----------|----------|
| 目标药物库存为0 | 不执行分拣，提示"该药品已缺货"，触发告警 |
| OCR/条码核验失败 | 暂停分拣，告警，等待管理员人工确认 |
| 抓取失败 (滑脱) | 重试最多3次，仍失败则标记为 failed |
| 药物已过期 | 拦截分拣，记录日志，通知管理员下架 |
| STM32通信超时 | 紧急停止，机械臂归位 |
| 库存低于阈值 | 自动推送补仓通知 |
