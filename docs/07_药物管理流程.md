## 1. 当前实现范围（单机械臂 + 固定药架）

由于硬件限制（单台 JetArm 六轴机械臂，无可移动底盘），系统当前聚焦于机械臂工作半径内的核心能力：

| 功能 | 状态 | 说明 |
|------|------|------|
| 药物分拣（语音→AI推荐→视觉定位→抓取→交付） | ✅ 实现中 | 核心闭环 |
| 固定药架药物定位与抓取 | ✅ 实现中 | 工作半径 ~350mm 内 |
| 视觉核验（YOLOv8 + OCR + 条码） | ✅ 实现中 | 三重校验 |
| 药物入库扫码登记 | 🟡 计划中 | 管理员手动扫码录入 |
| 自动化库存更新 | 🔮 未来 | 依赖移动底盘 |
| 全库补仓巡检与告警 | 🔮 未来 | 需 AMR 自主移动平台 |
| 过期药物自动检测 | 🔮 未来 | 需全库巡检能力 |

## 2. 当前药物分拣流程

```
Step 1: 接收分拣指令
  → 来源: AI Agent 药物推荐结果
  → 数据: {drug_id, drug_name, quantity}

Step 2: 位置检索
  → drug_id → 查询药物知识库 → shelf_location (固定货架坐标)
  → 生成目标位姿 (x, y, z, pitch)

Step 3: 视觉核验
  → 摄像头拍摄目标货架区域
  → YOLOv8 定位药盒
  → PaddleOCR 读取药名验证
  → pyzbar 扫描条码二次核验
  → 核验通过 → 计算3D位姿
  → 核验失败 → 告警，等待人工介入

Step 4: 抓取执行
  → 3D位姿 → ROS MoveIt 逆解
  → 六轴轨迹生成
  → STM32 舵机驱动
  → 夹爪抓取

Step 5: 交付
  → 搬运到取药窗口
  → 机械臂归位
  → TTS播报: "您的药物已备好，请取药"

Step 6: 手动入库补充
  → 管理员用条码扫描器登记药物
  → pyzbar 解码 → 写入库存表
  → 机械臂不参与入库操作（这是当前限制）
```

### 异常处理

| 异常场景 | 处理策略 |
|----------|----------|
| 目标药物库存为0 | 不执行分拣，提示"该药品已缺货，请联系药师" |
| 视觉核验失败 | 暂停分拣，告警，等待管理员人工确认 |
| 抓取失败（滑脱） | 重试最多3次，仍失败则标记为 failed |
| 药物不在工作半径内 | 提示管理员手动取药 |
| STM32通信超时 | 紧急停止，机械臂归位 |

## 3. 药物知识库管理

当前采用本地 `drugs.json` + ChromaDB 向量检索：

```json
{
  "drug_id": "DRG-001",
  "name": "阿莫西林胶囊",
  "shelf_location": "A-03-12",
  "stock_quantity": 50,
  "category": "抗生素/青霉素类"
}
```

库存表字段设计（预留，当前手动维护）:

| 字段 | 类型 | 说明 |
|------|------|------|
| drug_id | VARCHAR(32) PK | 药物唯一编号 |
| name | VARCHAR(128) | 药品名称 |
| stock_quantity | INT | 当前库存数量 |
| shelf_location | VARCHAR(16) | 货架位置 |
| alert_threshold | INT | 补仓告警阈值（预留） |
| expiry_date | DATE | 有效期（预留） |

## 4. 🔮 未来扩展方向

以下功能受限于当前硬件（单固定机械臂），需要后续接入自主移动底盘(AMR)实现：

### 4.1 全库药物巡检与自动补仓

```
需求: 机械臂安装到 AMR 上，自主导航到各货架区域
新增硬件: 
  - 自主移动底盘（如差速轮/麦克纳姆轮底盘）
  - 激光雷达或深度相机（SLAM导航）
  - 无线充电底座

新增软件:
  - ROS Navigation Stack (SLAM + 路径规划)
  - 多货架位姿注册与标定
  - 补仓巡检定时任务

流程:
  AMR巡航 → 到达目标货架 → 摄像头拍照 → OCR盘点库存
  → 低于阈值 → 自动从补货区取药 → 补入对应货架
```

### 4.2 过期药物自动检测

```
需求: 机械臂 + AMR 定期全库巡检
新增能力:
  - OCR读取有效期文字
  - 自动对比当前日期
  - 过期药物自动下架 → 移至待销毁区
```

### 4.3 库存实时看板

```
需求: 每次分拣后自动更新库存数量
当前限制: 手动维护库存表
未来方案:
  - 每次抓取动作联动库存计数
  - WebSocket 推送库存变更到管理后台
  - 低于阈值 → 管理后台弹窗提醒 + 邮件/短信推送
```

### 4.4 扩展路线图

```
Phase 1 (当前):  固定机械臂 + 语音/视觉/AI → 单站药物分拣
Phase 2 (未来):  + AMR移动底盘 → 多货架自主取药
Phase 3 (未来):  + 库存自动盘点 → 补仓巡检自动化
Phase 4 (未来):  + 多机械臂协同 → 药房全自动化
```
