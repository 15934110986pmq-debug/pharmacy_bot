# Voice — 语音交互方案

> 版本: v1.0  |  日期: 2025-05-29

---

## 1. 概述

语音交互模块基于科大讯飞离线 SDK (`xf_mic_asr_offline`)，实现全离线的中文语音识别与 TTS 播报。患者可直接用中文描述症状（如"我头痛、发烧两天了"），系统自动识别并送入 AI Agent 进行诊断。

---

## 2. 语音管道架构

```
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│ 环形麦克风阵列 │────→│  讯飞离线ASR  │────→│  结构化输出    │
│ 六路波束成形   │     │  xf_mic_asr   │     │   文本/语义    │
└───────────────┘     └───────┬───────┘     └───────┬───────┘
                              │                      │
                              │                      ▼
                              │             ┌───────────────┐
                              │             │   AI Agent    │
                              │             │  症状分析+推荐 │
                              │             └───────┬───────┘
                              │                      │
                              │                      ▼
                              │             ┌───────────────┐
                              └─────────────│  TTS 语音播报  │
                                            │  药物反馈信息  │
                                            └───────────────┘
```

## 3. 讯飞离线 ASR (xf_mic_asr_offline)

### 3.1 技术规格

| 参数 | 值 |
|------|-----|
| 采样率 | 16kHz / 8kHz |
| 编码 | 16bit PCM |
| 帧长 | 640 samples (40ms@16kHz) |
| 缓冲区 | 4096 bytes |
| 唤醒词 | 可自定义语法文件 (call.bnf) |
| 声源定位 | 360° 波束成形 |
| 接口 | USB HID (通过 hidapi 驱动) |
| 离线模型 | ASR 模型 (`common.jet`, ~9.3MB)、TTS 模型 (小燕/小峰) |

### 3.2 关键数据结构 (C SDK)

```c
// 识别结果 (asr_offline_record_sample.h)
typedef struct _Recognise_Result {
    bool whether_recognised;    // 是否识别成功
    char *whole_content;        // 识别到的全部内容
    char fail_reason[32];       // 失败原因
} Recognise_Result;

typedef struct _Effective_Result {
    char effective_word[32];    // 识别到的有效文本
    int effective_confidence;   // 置信度 (0-100)
} Effective_Result;
```

### 3.3 Python 封装

现有 Python 桥接代码在 `voice_play.py`：

```python
# src/xf_mic_asr_offline/build/lib/xf_mic_asr_offline/voice_play.py
from xf_mic_asr_offline import xf_mic_asr
# 唤醒词检测 → 语音录制 → ASR 识别 → 文本返回
```

## 4. 语音交互流程

```
Step 1: 待机监听
  → 环形麦克风阵列持续监听
  → 唤醒词检测（如"你好药房"）
  → 声源定位确定说话人方向

Step 2: 语音录制
  → 唤醒后开始录音 (最长 10s)
  → 16kHz PCM 实时送入 ASR 引擎

Step 3: 离线识别
  → 讯飞离线引擎解码
  → 语法优化（自定义药学相关词库）
  → 输出识别文本 + 置信度

Step 4: 结果传递
  → 低置信度 (< 60): TTS提示"请再说一遍"
  → 高置信度 (≥ 60): 文本送入AI Agent

Step 5: AI推理 & 播报
  → AI Agent 处理，输出药物推荐
  → TTS 将推荐结果合成语音
  → 扬声器播报: "根据您的症状，推荐..."
```

## 5. 药物领域定制

### 5.1 自定义语法 (BNF)

在 `call.bnf` 中添加药物相关词组：

```
#src/xf_mic_asr_offline/config/call.bnf
<command>: 我要拿药 | 请帮我找一下 <drug_name> | 我< symptom >了 <duration>;

<drug_name>: 阿莫西林 | 布洛芬 | 头孢 | 对乙酰氨基酚;
<symptom>: 头疼 | 发烧 | 咳嗽 | 流鼻涕 | 肚子疼;
<duration>: 一天 | 两天 | 三天 | 一周;
```

### 5.2 热词增强

针对药品名称等低频词，利用讯飞热词接口提升识别准确率。

## 6. TTS 语音播报

### 内置音色

| 音色 | 发音人 | 风格 |
|------|--------|------|
| xiaoyan.jet (6.2MB) | 小燕 | 亲切女声 |
| xiaofeng.jet (6.2MB) | 小峰 | 稳重男声 |
| common.jet (20.4MB) | 通用合成 | - |
| common_en.jet (3.1MB) | 英语合成 | - |

### 播报内容模板

```
"您好，根据您的症状 [症状描述]，推荐使用 [药品名称]，
 用法为 [用法用量]。请注意 [注意事项/禁忌]。"
```
