# AI Agent — 云端API智能诊断与药物推荐系统

> 版本: v2.0  |  日期: 2025-05-29  |  更新: 改用云端 LLM API

---

## 1. 概述

AI 智能体接收用户的症状描述，**通过 API 调用云端大语言模型**完成症状分析、鉴别诊断和药物推荐。本地 Jetson Orin NX 不运行大模型推理，只负责 API 调用、Prompt 构造、检索结果注入和结果解析。部署方式简洁、不消耗边缘 GPU 算力。

## 3. 为什么用 API 而不是本地部署？

| 对比维度 | 本地 Ollama | 云端 API |
|----------|-----------|---------|
| Jetson GPU 占用 | 4-8GB 显存 (7B模型) | 0 |
| 推理速度 | 5-10 t/s (7B) | 50+ t/s |
| 模型版本 | 固定，需手动更新 | 始终最新 |
| 运维成本 | 需维护模型文件 | 零运维 |
| 网络依赖 | 无 | 需要公网 |
| 成本 | 硬件折旧 | API 按 token 计费 (~¥0.001/1K tokens) |

**结论**: 药房场景通常有 WiFi/有线网络，API 方式更合适。云端模型更大（如 DeepSeek-V3 671B vs 本地 7B），诊断质量显著更高。

## 6. 推理流程

```
Step 1: 症状接收
  → 来源: 语音识别文本 / 文字输入

Step 2: RAG 本地检索
  → 症状向量化 → ChromaDB 检索 Top-5 匹配药物
  → 组装 RAG Context

Step 3: API 调用
  → 构造 Prompt (System + Context + User)
  → HTTP POST → 云端 LLM
  → 等待返回 (~1-3s)

Step 4: 响应解析
  → 解析 JSON → 校验字段完整性
  → 药物 ID 映射到本地知识库(location, stock)

Step 5: 禁忌二次检测 (本地)
  → 推荐的药物是否有交互禁忌
  → 是否有过敏史冲突

Step 6: 输出
  → TTS 播报
  → 药物ID → 下发给分拣系统
```

### 性能预估

| 步骤 | 时延 |
|------|------|
| RAG 本地检索 | ~0.5s |
| API 网络往返 | ~0.3s |
| LLM 推理 (云端) | ~1-3s |
| 响应解析 + 禁忌检测 | ~0.2s |
| **总计** | **~2-4s** |

对比本地 Ollama (10-15s)，快了 3-5x。

### llm_client.py

```python
"""多Provider LLM API客户端 — OpenAI SDK兼容"""
import os, json
from openai import OpenAI

PROVIDERS = {
    "deepseek": {
        "base_url": "https://api.deepseek.com/v1",
        "model": "deepseek-chat",
        "env_key": "DEEPSEEK_API_KEY"
    },
    "qwen": {
        "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
        "model": "qwen-turbo",
        "env_key": "DASHSCOPE_API_KEY"
    }
}

class LLMClient:
    def __init__(self, provider="deepseek"):
        cfg = PROVIDERS[provider]
        self.client = OpenAI(
            api_key=os.getenv(cfg["env_key"]),
            base_url=cfg["base_url"]
        )
        self.model = cfg["model"]

    def chat(self, system_prompt, user_input, context="", json_mode=True):
        messages = [{"role": "system", "content": system_prompt}]
        if context:
            messages.append({"role": "assistant", "content": f"[Context]\n{context}"})
        messages.append({"role": "user", "content": user_input})

        kwargs = {
            "model": self.model,
            "messages": messages,
            "temperature": 0.3,
            "max_tokens": 2048
        }
        if json_mode:
            kwargs["response_format"] = {"type": "json_object"}

        resp = self.client.chat.completions.create(**kwargs)
        content = resp.choices[0].message.content
        return json.loads(content) if json_mode else content
```

### drug_kb.py

```python
"""药物知识库 — ChromaDB RAG检索"""
import chromadb, json
from chromadb.utils import embedding_functions
from typing import List

class DrugKnowledgeBase:
    def __init__(self, db_path="./drug_kb_store"):
        self.client = chromadb.PersistentClient(path=db_path)
        self.ef = embedding_functions.DefaultEmbeddingFunction()
        self.collection = self.client.get_or_create_collection(
            name="drugs", embedding_function=self.ef
        )

    def load_drugs(self, drugs_json_path):
        """从JSON加载药物数据，构建向量索引"""
        with open(drugs_json_path) as f:
            drugs = json.load(f)
        for drug in drugs:
            doc = f"{drug['name']} {drug['generic_name']} "
            doc += f"适应症: {' '.join(drug['indications'])} "
            doc += f"禁忌: {' '.join(drug.get('contraindications', []))}"
            self.collection.add(
                ids=[drug["drug_id"]],
                documents=[doc],
                metadatas=[drug]
            )

    def retrieve(self, query: str, k: int = 5) -> List[dict]:
        """症状 → Top-K 匹配药物"""
        results = self.collection.query(query_texts=[query], n_results=k)
        if not results["ids"][0]:
            return []
        return [
            {
                "drug_id": mid,
                "drug_name": meta["name"],
                "indications": meta.get("indications", []),
                "contraindications": meta.get("contraindications", []),
                "dosage": meta.get("dosage", {}),
                "shelf_location": meta.get("shelf_location", "")
            }
            for mid, meta in zip(results["ids"][0], results["metadatas"][0])
        ]
```

### symptom_agent.py

```python
"""症状诊断Agent — 编排RAG检索 + API调用 + 禁忌检测"""
import json
from .llm_client import LLMClient
from .drug_kb import DrugKnowledgeBase

SYSTEM_PROMPT = """你是资深药剂师。根据患者症状和药物知识库检索结果给出推荐。
输出严格JSON: {"diagnosis":"","recommendations":[],"warnings":[],"follow_up":""}"""

class SymptomAgent:
    def __init__(self, provider="deepseek", kb_path="./drug_kb_store"):
        self.llm = LLMClient(provider=provider)
        self.kb = DrugKnowledgeBase(kb_path)

    def diagnose(self, symptom_text: str) -> dict:
        # 1. RAG检索
        candidates = self.kb.retrieve(symptom_text, k=5)
        context = json.dumps(candidates, ensure_ascii=False, indent=2)

        # 2. API推理
        result = self.llm.chat(SYSTEM_PROMPT, symptom_text, context)

        # 3. 禁忌检测（本地）
        result = self._check_contraindications(result)

        return result

    def _check_contraindications(self, result: dict) -> dict:
        # TODO: 交叉检查推荐药物的禁忌是否冲突
        return result
```

## 9. 注意事项

1. **网络容错**: API 调用失败时重试3次，超时设置 15s，失败时 Fallback 到本地规则引擎
2. **隐私**: 症状数据脱敏后再发送到云端（移除姓名等 PII）
3. **成本控制**: DeepSeek chat ~¥1/百万tokens，单次诊断约 ¥0.002，月均 ¥30-50（按日均 50 次诊断）
4. **离线降级**: 网络不可用时，降级为本地规则引擎匹配
