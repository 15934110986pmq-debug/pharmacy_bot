# AI Agent — 智能诊断与药物推荐系统

> 版本: v1.0  |  日期: 2025-05-29

---

## 1. 概述

AI 智能体是本项目的核心决策模块。它接收用户的症状描述（来自语音识别或文字输入），利用本地部署的大语言模型和药物知识库，完成症状分析、鉴别诊断、药物推荐、交互禁忌检测和推荐理由生成。

---

## 3. 核心模块设计

### 3.1 药物知识库 (Drug KB)

#### 数据模型

```json
{
  "drug_id": "DRG-001",
  "name": "阿莫西林胶囊",
  "generic_name": "Amoxicillin",
  "category": "抗生素/青霉素类",
  "indications": [
    "上呼吸道感染",
    "急性扁桃体炎",
    "中耳炎"
  ],
  "contraindications": [
    "青霉素过敏者禁用",
    "肾功能不全者慎用"
  ],
  "interactions": [
    "与甲氨蝶呤合用时毒性增加",
    "与口服避孕药合用降低避孕效果"
  ],
  "dosage": {
    "adult": "0.5g，每日3次",
    "child": "20-40mg/kg/日，分3次"
  },
  "shelf_location": "A-03-12",
  "stock_quantity": 50,
  "alert_threshold": 10,
  "expiry_date": "2027-06-15"
}
```

#### RAG 检索流程

```
用户症状 → 向量化 (embedding model)
  → ChromaDB 检索 Top-K 匹配药物
  → 重排序 (基于症状匹配度 + 禁忌检测)
  → 将检索结果注入 LLM Prompt
```

### 3.2 LLM 推理

#### 模型选型

| 模型 | 大小 | 用途 | 理由 |
|------|------|------|------|
| Qwen2.5 (7B) | ~4.4GB | 多轮对话、症状追问 | 中文能力强，支持医药领域 |
| DeepSeek-Coder (6.7B) | ~4.0GB | 结构化推荐输出 JSON | 结构化输出精准 |

Ollama 已在 Jetson 上预装（见 `7.主板文件/Jetson Orin NX 主板/第9章 AI大模型课程/`）。

#### Prompt 设计

```
System: 你是一位资深药剂师和临床医学顾问。根据患者描述的症状，
        检索药物知识库中的相关信息，给出专业的药物推荐。
        输出格式为严格JSON：
        {
          "diagnosis": "初步判断",
          "recommendations": [{"drug_id": "", "drug_name": "", ...}],
          "warnings": ["禁忌1", "禁忌2"],
          "advice": "其他建议"
        }

Context: [RAG检索结果: 相关药物的适应症、禁忌、用法用量...]

User: [患者症状描述]
```

### 3.3 症状抽取 (NER)

利用 LLM 的 few-shot 能力抽取结构化症状实体：

```
输入: "我头痛两天了，还有点发烧，咳嗽得很厉害"
输出:
{
  "symptoms": [
    {"name": "头痛", "onset": "2天前"},
    {"name": "发烧", "degree": "未描述"},
    {"name": "咳嗽", "severity": "剧烈"}
  ],
  "patient_info": {
    "age": "未提供",
    "allergies": "未提供"
  },
  "missing_info": ["体温", "年龄", "药物过敏史", "既往病史"]
}
```

### 3.4 禁忌检测

在推荐药物后，自动检查：

1. **药物-过敏禁忌**: 如患者青霉素过敏 → 排除青霉素类药物
2. **药物-药物交互**: 如患者已在服用药A → 避免与A有交互的药B
3. **药物-剂量校验**: 超量/不足时发出警告

## 4. 推理流程

```
Step 1: 症状接收
  → 来源: 语音识别文本 / 手动输入文字
  → 预处理: 去除口语化噪声

Step 2: 症状抽取
  → LLM few-shot 抽取结构化症状
  → 识别缺失信息，主动追问

Step 3: RAG 检索
  → 症状向量化
  → ChromaDB 检索 Top-K (K=5) 匹配药物
  → 编码禁忌约束，初步过滤明显不适用药物

Step 4: LLM 推理
  → Prompt = System(角色) + Context(RAG结果) + User(症状)
  → Qwen2.5 推理
  → DeepSeek-Coder 校验输出格式

Step 5: 后处理
  → 禁忌二次检测
  → 格式化为结构化 JSON
  → 生成推荐理由 (不可解释的LLM输出需人工审核)

Step 6: 输出
  → TTS 播报推荐结果
  → JSON 下发到药物分拣系统
  → 操作日志记录
```

## 5. 部署方案

### Jetson Orin NX 部署

```bash
# 安装 Ollama (参考主板文档第9章)
curl -fsSL https://ollama.com/install.sh | sh

# 拉取模型
ollama pull qwen2.5:7b
ollama pull deepseek-coder:6.7b

# 验证
ollama run qwen2.5:7b "什么是阿莫西林的适应症？"
```

### 性能预估

| 步骤 | 时延 (Jetson Orin NX) |
|------|----------------------|
| 症状抽取 (NER) | ~2s |
| RAG 检索 (ChromaDB) | ~1s |
| LLM 推理 (Qwen2.5 7B) | ~5-10s |
| 后处理+禁忌检测 | ~0.5s |
| **总计** | **~10-15s** |
